# Параграф — сайт-витрина

Статический сайт бренда «Параграф» (правовые документы для онлайн-репетиторов):
витрина + захват в Telegram + обязательная публикация правовых документов по 152-ФЗ.

Без сборки и без Node — чистый HTML + CSS + ванильный JS. Живой фон, плавающие «§»,
параллакс, scroll-анимации, счётчик штрафа, FAQ-аккордеон, мобильное меню.

---

## 1. Как посмотреть локально

**Вариант А (рекомендую) — мини-сервер на PowerShell:**
правый клик по `serve.ps1` → «Запустить с помощью PowerShell».
Откроется `http://localhost:8080`. Остановить — `Ctrl+C`.

> Если PowerShell ругается на политику запуска, выполните один раз:
> `powershell -ExecutionPolicy Bypass -File .\serve.ps1`

**Вариант Б — просто двойной клик** по `index.html`.
Почти всё работает и так; сервер из варианта А аккуратнее отдаёт пути и шрифты.

## 3. Требования 152-ФЗ — что уже заложено

- Ссылка на **Политику** видна в футере на **каждой** странице (политика «опубликована»).
- Форма-заявка на главной: **два чекбокса** (согласие на ПДн + принятие политики) сняты
  по умолчанию, кнопка «Отправить» неактивна, пока не отмечены **оба**.
- ПДн на сайте **не хранятся**: заявка открывает Telegram-бота (сбор данных — там).
- В Политике есть все обязательные разделы (оператор, цели, основания, перечень ПДн,
  действия и способы, сроки, права и отзыв, трансграничная передача, локализация, безопасность).
- Cookie и веб-аналитика (Яндекс.Метрика): есть **cookie-баннер** на каждой странице и
  **Политика cookie** (`cookie.html`). Метрика подключается только после согласия — вставьте
  код счётчика в `initAnalytics()` (`assets/js/main.js`).

## 4. Деплой (выберите одно)

Сайт статический — подойдёт любой хостинг статики.

**Vercel / Netlify (drag-and-drop):** зайдите на сайт сервиса → перетащите папку
проекта целиком → готово. Build-команда не нужна, output — корень проекта.

**GitHub Pages:**
```powershell
git init
git add .
git commit -m "Параграф — сайт"
git branch -M main
git remote add origin https://github.com/<вы>/paragraf-site.git
git push -u origin main
```
Затем в репозитории: Settings → Pages → Branch `main` / `root`. Файл `.nojekyll` уже добавлен.

> Домен желательно **.ru**. Поскольку формы сбора ПДн на сайте нет (сбор только в боте),
> требования к локализации к статике мягче, но **Политику публиковать обязательно**.
> После публикации обновите `{{ДОМЕН}}` и пути в `og:image`, `canonical`, `sitemap.xml`.

## 5. Структура

```
index.html        — главная (hero, боль, цены, для кого, шаги, юр-блок, отзывы, лид, FAQ, CTA)
privacy.html      — Политика конфиденциальности (реальный текст)
consent.html      — Согласие на обработку ПДн (реальный текст)
offer.html        — Публичная оферта (реальный текст)
cookie.html       — Политика использования файлов cookie (реальный текст)
assets/css/styles.css   — дизайн-система, компоненты, анимации, адаптив
assets/js/main.js       — интерактив: курсор, магнит-кнопки, тилт, reveal, счётчик, форма
assets/js/hero3d.js     — 3D-сцена hero (Three.js): парящие документы, золотые печати, пыль
assets/img/             — favicon (§), avatar, og-cover
serve.ps1         — локальный предпросмотр без Node
robots.txt, sitemap.xml, .nojekyll
```

> **3D-сцена.** `hero3d.js` подгружает Three.js с CDN (jsdelivr) через `importmap` в `index.html`.
> Нужен интернет; при его отсутствии сцена просто не появляется — сайт работает как обычно.
> Сцена отключается на мобильных (<760px) и при `prefers-reduced-motion`. Чтобы убрать 3D —
> удалите `<div id="hero-3d">`, importmap и `<script ... hero3d.js>` из `index.html`.
> Для офлайн/самохостинга скачайте `three.module.js` и папку `examples/jsm` локально и
> поправьте пути в `importmap`.
